Command Line Options

The Equinor petro-elastic modelling tool. Models elastic properties of sandstone saturated with oil, gas, and brine. See github.com/equinor/open_petro_elastic for documentation.

usage: open_petro_elastic [-h] [--data-file DATA_FILE]
                          [--calibration CALIBRATION_DATA]
                          [--output_file OUTPUT_FILE] [--version]
                          [--generate-tutorial GENERATE_TUTORIAL]
                          [--success_threshold SUCCESS_THRESHOLD]
                          config_file

Positional Arguments

config_file

petro elastic yaml input file

Named Arguments

--data-file

Optional additional csv file for data to be inserted into config.

--calibration

Optional calibration data, if given, an error statistic is calculated and printed.

--output_file

Optional output file, defaults to stdout.

Default: <_io.TextIOWrapper name=’<stdout>’ mode=’w’ encoding=’utf-8’>

--version

show program’s version number and exit

--generate-tutorial

Write tutorial config. Takes directory to write to as argument.

--success_threshold

Optional percentage required for success. Default is 100 percent with no partial results given. Unsuccessful rows will be given NaN. Note that this option can deteriorate performance.
